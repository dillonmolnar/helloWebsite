What’s here

  • index.html — this page
  • styles.css — layout and theme
  • js/cognito-config.js + js/cognito-client.js — Hosted UI + PKCE
  • callback.html — OAuth redirect handler

No build step required. Amplify can serve these files as-is from the repository root.

Quick deploy checklist

  1. Initialize git in this folder and push to GitHub.
  2. In AWS Amplify: Host web app → connect GitHub.
  3. Use the default build settings for a static site (or add a minimal amplify.yml if prompted).
  4. Save and deploy; your URL appears when the first build finishes.

Amazon Cognito (Hosted UI)

This site uses a User Pool app client (public, no secret) and the Hosted UI with authorization code + PKCE. After you create resources in AWS, paste values into js/cognito-config.js and redeploy.

  1. In Cognito: create a User pool (or use an existing one) with at least email sign-in.
  2. Under App integration → Domain, set a Cognito domain prefix (e.g. myapp-auth). That value is domainPrefix in config.
  3. Under App clients, add a client: public (no secret), allowed flows: Authorization code grant. OAuth 2.0 scopes: openid, email, profile.
  4. In the same app client, set Allowed callback URLs to your live callback.html URL (example: https://main.d123.amplifyapp.com/callback.html).
  5. Set Allowed sign-out URLs to your site root (example: https://main.d123.amplifyapp.com/) — must match signOutRedirectUri exactly.
  6. Copy the app client id and region into js/cognito-config.js, push to GitHub, and test Sign in in the header.

For social IdPs (Google, etc.), add them under Sign-in experience → Federated identity providers, then enable them on the Hosted UI. For API authorization, use the access token with API Gateway Cognito authorizers or your backend.
